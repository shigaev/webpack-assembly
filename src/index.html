About

Конфигурация:

Кофигурация - это настройка вебпака, которая говорит откуда файл брать, куда его положить и как преобразовать, если это необхдимо.

Точка входа:

Вебпак строит граф зависимостей.

Он проходится по всем импортам в файлах, строит граф и загружает их в финальный файл сборки.

Входная точка - файл, с которого вебпак начинает построение графа. Этот файл нужно включить первым, от него будут идти все импорты. Обычно это index.js

Точка выхода:

Вебпак отдает нам один файл, этот файл и является точкой выхода. Обычно находится в папке dist

Лоадеры:

Вебпак понимает только JavaScript и JSON. Если мы попытаемся запихнуть внутрь другие типы файлов (html, css, png, jpg, typescript и т.д.) то вебпак выдаст ошибку.

Для того чтобы такого не происходило и придуманы лоадеры.

Лоадеры - это специальные модули, для того чтобы считывать и обрабатывать файлы. Для них в свою очередь придуманы парвила

Правила: - это в свою очередь объекты в кофигурации, которые указывают на файл, который нужно обработать и указывают лоадер, которым этот файл нужно обработать.

Плагины:

Лоадеры просто обрабатывают файлы, а плагины используются для того чтобы управлять не только импортами в JS.

Плагины: - внешние модули для webpack, которые позволяют управлять и обрабатывать файлы, которые не импортируются в JavaScript.

Режим:

У вебпака есть три режима:

  • development
  • production
  • отсутсвие режима
